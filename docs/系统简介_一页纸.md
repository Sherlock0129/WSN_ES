# 无线传感器网络能量传输仿真系统 - 一页纸简介

## 🎯 系统核心功能

**本系统做什么？**
> 模拟无线传感器网络中的能量传输过程，研究如何通过无线能量传输（WET）延长网络寿命、实现能量均衡。

## 🔑 核心概念（3分钟理解）

### 问题背景
无线传感器节点能量有限 → 部分节点死亡 → 网络功能丧失

### 解决方案
让**能量充足的节点**向**能量不足的节点**无线传输能量 → 延长网络寿命

### 系统角色
```
系统 = 实验平台 + 算法测试床
     ├─ 创建虚拟WSN网络（10-100个节点）
     ├─ 模拟能量采集（太阳能）
     ├─ 测试不同调度算法
     └─ 评估性能指标
```

## 🏗️ 三大核心模块

### 1️⃣ 网络仿真
- **功能**：创建虚拟传感器网络
- **包含**：节点能量、位置、通信能力
- **支持**：太阳能采集、能量衰减、移动节点

### 2️⃣ 能量调度
- **功能**：决策哪些节点之间传输能量
- **算法**：
  - 传统：Lyapunov优化
  - AI：DQN/DDPG深度强化学习
- **目标**：能量均衡 + 高效传输

### 3️⃣ 路径规划
- **功能**：为能量传输规划最优路径
- **协议**：EETOR（能量传输专用路由）
- **考虑**：传输效率、多跳损耗、节点能量

## 📊 典型仿真流程

```
初始化
  ↓
创建15个传感器节点（部分有太阳能）
  ↓
仿真7天（10080分钟）
  ↓
每个时间步：
  ├─ 能量更新（采集、衰减、消耗）
  ├─ 决策是否传能（智能被动触发）
  ├─ 如果需要：
  │   ├─ 调度器选择donor/receiver
  │   ├─ 路由器规划传输路径
  │   ├─ 执行能量传输
  │   └─ 记录统计数据
  └─ 继续下一步
  ↓
输出结果和性能报告
```

## 🎓 支持的调度算法

### 传统优化
- ✅ **LyapunovScheduler** - 基于优化理论
- ✅ **DurationAwareScheduler** - 时长感知优化
- ✅ **AdaptiveDurationScheduler** - 自适应时长

### 深度学习
- ✅ **DQNScheduler** - 离散动作（1-10分钟）
- ✅ **DDPGScheduler** - 连续动作（精确控制）

## 🔧 关键配置

**在 `src/config/simulation_config.py` 中配置：**

```python
# 基础配置
num_nodes = 15                    # 节点数量
time_steps = 10080                # 仿真时长（分钟）
solar_ratio = 0.6                 # 太阳能节点比例

# 调度器选择
scheduler_type = "lyapunov"       # 或 "dqn", "ddpg"

# 智能被动传能
passive_mode = True               # 启用智能触发
check_interval = 10               # 检查间隔（分钟）

# DQN训练（可选）
enable_dqn = True
dqn_training_mode = True          # 训练模式
dqn_training_episodes = 50        # 训练回合数
```
