主要差别对比
============================================================

1. 决策依据
   原来: 使用 info_volume (信息量)
   现在: 使用 info_value (信息价值 = 信息量 x 时间衰减因子)

2. 延迟上报决策
   原来: info_volume >= threshold -> 等待
   现在: info_value >= threshold -> 等待
   影响: 等待时间长的信息会更快上报

3. 路由选择
   原来: 优先选择 info_volume 大的节点
   现在: 优先选择 info_value 高的节点
   影响: 自动优先选择信息更新鲜的节点

4. 调度器决策
   原来: 使用 info_volume > 0 判断
   现在: 使用 info_value > 0 判断
   影响: 等待时间长的信息可能被视为无价值
